# Gooda Design Guidelines

本文档用于 1.0 编辑器维护和 2.0 Dimina 迁移期间保持一致的视觉、美工和 UI/UX 风格。它不是组件 API 文档，而是风格边界与设计决策清单；实现细节仍以 `src/styles/gooda-theme.css` 和页面组件为准。

## 1. Design Intent

谷搭是面向痛包/谷子摆放的手机端编辑器。第一屏应直接是可编辑画布，不做营销式落地页；用户打开后看到的是自己的痛包、素材抽屉和必要编辑工具。

整体气质是“水彩纸面上的柔软工具”：轻、透明、手作、偏蓝、带一点痛包/谷子文化的装饰感。它不是重型生产力软件，也不是高对比深色工具；功能要足够密，但视觉上要像在纸面和贴纸上操作。

## 2. Visual Language

> **Claude Design 蓝白简约 refresh（当前方向，覆盖下方部分旧玻璃约定）**
> 在保留水彩背景 + 手作气质的前提下，弹层里的**内容面板**（资料库、图层、导入编辑）走「扁平蓝白」：
> - 关键主操作（搜索 / 导入 / 确认 / 保存）用**实心蓝胶囊**（`--blue-strong`），白字，一眼可见——不再是浅蓝玻璃底。
> - 列表用 **hairline 分割**替代逐行卡片底；表单字段用 hairline 分隔的节奏替代嵌套圆角描边容器。
> - 选项（类型 / 选区）未选=白底 hairline 描边，选中=实心蓝。
> - 卡片=扁平白卡（白底 + hairline 内环 + 软阴影），不做分层玻璃。
> - 玻璃只保留给**浮动工具胶囊 / 遮罩弹层容器**，不再铺满内容面板。
> 落地范围：`.spu-*` / `.layer-*`+`.row-*` / `.import-*`。

核心关键词：

- 水彩纸纹背景：背景是产品气质的一部分，不要换成纯色、深色渐变或抽象科技背景。
- iOS glass：浮动工具胶囊、遮罩弹层容器、抽屉用半透明白、低饱和蓝、轻 blur、内高光；内容面板走上方蓝白简约。
- 水蓝主色：蓝色用于状态、强调、选中；关键主操作用实心蓝胶囊，其余处不做大面积实心填充。
- 手写/宋体系文字：大类 Tab、部分按钮和标签偏手作感；系统信息和编辑数值保持清晰。
- 轻边框、低阴影：阴影是空气感，不是卡片悬浮感。
- 真实素材优先：痛包、底板、谷子、导出预览必须尽量显示真实图，不用抽象占位替代真实外观。

## 3. Tokens To Preserve

当前主要 token 位于 `src/styles/gooda-theme.css` 的 `.page` 下。迁移时优先复用这些值，避免新增相近但不一致的蓝色、玻璃色或阴影。

```css
--blue-strong: #1f65b7;
--blue-soft: #5f95c7;
--blue-bg: #eef7ff;
--text-strong: #183d70;
--text-primary: #1d1d1f;
--glass-panel: rgba(255,255,255,.18);
--glass-panel-strong: rgba(255,255,255,.28);
--glass-panel-border: rgba(255,255,255,.30);
--glass-control: rgba(255,255,255,.34);
--glass-control-soft: rgba(255,255,255,.18);
--font-serif: "Songti SC", "STSong", "Noto Serif CJK SC", "Times New Roman", serif;
--shadow-blue-soft: 0 7PX 17PX rgba(31,101,183,.045);
--motion-spring: cubic-bezier(.18,.89,.32,1.08);
```

### Color Rules

- 主蓝 `#1f65b7` 只用于关键状态和重点按钮，不要让页面变成大片蓝色 UI。
- 图标黑灰优先使用近似 `#34363a` / 低饱和滤镜，不用纯黑大面积压住水彩背景。
- 警告/危险操作也保持柔和，不要突然使用高饱和红色；如果需要红色，做轻量局部提示。
- 不新增紫色、深蓝、米色、棕色作为主色系。

### Typography Rules

- 页面基础字体使用 Apple/system sans，保证工具读数清晰。
- 大类 Tab、素材名、弹层标题可用 `--font-serif`，保持手作/纸面气质。
- 不使用 viewport-width 缩放字体。
- 不使用负字距。
- 紧凑控件中的文字必须小而稳，不要用 hero 级字号。

## 4. Layout Principles

### Full Screen Editor

- 页面是固定手机编辑器：背景全屏，画布居中，顶部工具栏和底部抽屉围绕画布服务。
- 第一屏必须露出痛包主体，不能被顶部栏、原生胶囊、抽屉遮住关键区域。
- 画布、痛包、底板和素材应保持 WYSIWYG：编辑态、预览卡片、导出结果尽量一致。

### Top Bar

- 顶部栏只承载 logo、undo/redo、导出历史、导出。
- 顶部按钮整体需要在左侧 logo 与右侧小程序原生胶囊之间排布，保留原生按钮安全区。
- 顶部按钮是小玻璃按钮：约 31PX 高，圆角约 10PX，图标约 20PX，低饱和黑灰。
- 禁止回到纯文字“保存/导出”大按钮样式；保存功能不是主入口。

### Stage Tool Rail

- 左侧浮动工具栏是玻璃胶囊，纵向排列 zoom/grid/center/layer 等工具。
- 抽屉收起时工具栏可滑出屏幕并淡出；抽屉展开时弹入。
- 可拖动浮窗不要侵入顶部 logo/按钮保护区，也不要覆盖痛包主体太多。

### Bottom Drawer

- 底部素材抽屉是主要操作面板，使用大圆角玻璃容器。
- 收起态要露出大类 Tab，且把手位置在收起/展开之间保持一致感。
- 大类 Tab 使用手写图片、波浪下划线和小星光，表达性可以强。
- 二级分类要安静：胶囊、浅蓝选中态、横向滚动。
- 素材网格卡片为浅玻璃方格，真实素材缩略图完整可辨。

## 5. Component Style Rules

### Buttons

- 优先用图标按钮，不用文字包进圆角矩形，除非是明确命令或弹层内操作。
- 图标用现有 PNG/icon asset 或同风格透明底图标；不要混入明显不同线宽、不同黑度的图标。
- Disabled 用 opacity 降低，不要换成灰色实心块。
- Primary 操作（弹层内容里的搜索 / 导入 / 确认 / 保存等关键 CTA）用**实心蓝胶囊**（`--blue-strong`），白字；这是蓝白简约 refresh 的方向。顶部工具栏的图标按钮仍是小玻璃按钮，不受此影响。

### Cards And Panels

- 避免页面区块卡片套卡片。
- 素材卡片、历史条目、弹层项可以是卡片；页面主要区域不要做重浮层。
- 圆角标准：
  - 小按钮 9-10PX
  - 素材卡片约 15PX
  - 弹层约 22-24PX
  - 抽屉顶部大圆角可更大

### Selection Overlay

- 选中框保持矩形，不随圆形素材变圆。
- 虚线/实线都要细，不要粗蓝框。
- 四角操作按钮保持轻玻璃圆形，图标黑灰；按钮可以跟随选中框旋转。
- 移动中可隐藏把手，降低视觉噪音。

### Nudge Controls

- 微调胶囊条是辅助工具，不应抢视觉焦点。
- 数值使用 tabular nums；缩放与角度布局稳定，不因内容变化抖动。
- 角度微调最小单位当前为 1 度。

### Modals / Export / History

- 弹层背景使用轻 blur 和半透明深色遮罩，面板保持玻璃白。
- 导出预览必须展示完整导出图。
- 导出历史列表显示缩略图、名称、时间；点击后打开导出预览。

## 6. Motion And Interaction

- 动效要柔和、短、带弹性：常用 `.18s` 到 `.36s`。
- 抽屉展开/收起、浮动工具栏弹入/滑出、按钮状态都应有过渡。
- 拖动中禁用 transition，释放后再吸附或回弹。
- 交互优先级：
  - 选中素材时，背景单击优先取消选中。
  - 抽屉展开且未选中时，背景单击可收起抽屉。
  - 抽屉收起且未选中时，背景单击可拉起抽屉。
- 不要让抽屉手势与系统底部手势冲突；收起态高度要留足可点可拖区域。

## 7. Asset And Image Rules

- 真实痛包图、底板图、素材图优先于 SVG/抽象占位。
- 痛包预览卡片必须显示完整痛包，不只显示透明窗口/底板。
- 导入素材和裁剪预览必须与舞台放置效果一致。
- 图标素材需要透明底，并和现有按钮线宽、大小、黑灰程度一致。

## 8. 2.0 / Dimina Migration Notes

2.0 使用 Dimina 目标时，视觉一致优先，但不要为了复刻 1.0 写法而破坏渲染正确性。

必须保留的 Dimina 约束：

- 裁剪预览卡片使用 px box + px branch 的 `cropInnerStyle` / `cropBoxFitPx`，不要切回百分比定位。百分比 width/height/left/top 在嵌套 `<image>` 上会渲染错误区域。
- 底板缩略图使用 aspect-matched px box + `scaleToFill`，不要依赖 `<image aspectFill>` 的隐式居中。
- 2.0 的手动滚动素材网格是为了解决原生 scroll-view 与拖出手势冲突，不要轻易改回 native scroll。
- H5 可作为快速检查，但 Dimina 真实运行仍需单独验证；不要用 H5 通过代替真机/Dimina 通过。

## 9. Do / Don't

Do:

- 先复用 `gooda-theme.css` token。
- 继续使用水彩背景、浅玻璃、手写大类 Tab。
- 用真实素材图表达产品状态。
- 保持按钮小而清晰，控件尺寸稳定。
- 修改 UI 后至少运行对应构建检查。

Don't:

- 不做营销首页、hero、说明型首屏。
- 不引入深色工具台、纯黑重图标、厚重卡片阴影。
- 不把大类 Tab 改成普通文字 tabs。
- 不把所有蓝色都做成实心高饱和主按钮。
- 不把 2.0 的 Dimina 渲染约束回退成 1.0 的 H5 写法。

## 10. Verification Expectations

1.0 常用检查：

```bash
npm run build:h5
```

2.0 常用检查：

```bash
npm run check
npm run build:h5
```

2.0 还有 Dimina 相关验证脚本，详见 `src/pages/index/README.md`。涉及裁剪、导入、拖动、导出历史时，优先在真实目标环境确认。
